다음 절차를 정확히 따라줘. **사용자가 "예"라고 답하기 전까지 절대 `prompt.md`를 수정하지 마.**

## 1. 읽기

프로젝트 루트의 `prompt.md`를 Read 툴로 읽어. 비어있거나 공백만 있으면 "prompt.md가 비어있어요. 내용을 작성한 뒤 다시 `/claude_move`를 호출해주세요."라고 한 줄로 알리고 종료.

## 2. 정제

읽은 내용을 분석해서 **명확하고 구조화된 Claude Code 프롬프트**로 다시 써줘:
- 모호한 표현은 구체적으로
- 여러 작업이면 단계로 분리
- 파일 경로·조건·제약이 누락됐으면 합리적으로 추론하거나 `[?]`로 명시
- 너무 짧은 입력(한 줄짜리)은 의도를 파악할 수 있는 형태로 적절히 확장
- 입력 언어 유지 (한국어 입력 → 한국어 출력)
- 사용자가 명시한 단어·고유명사·파일명은 그대로 보존

정제된 프롬프트를 fenced code block(```)으로 보여줘.

## 3. 검토 요청

`AskUserQuestion` 툴을 호출해서:
- 질문: "이 프롬프트대로 진행할까요?"
- 헤더: "프롬프트 승인"
- 옵션:
  1. **"예, 진행해줘"** — 정제 결과대로 작업 시작
  2. **"아니오, 다시 다듬어줘"** — prompt.md 유지하고 수정 방향 받기

## 4. 분기

### 사용자가 "예"를 선택하면
1. Write 툴로 `prompt.md`의 내용을 빈 문자열(`""`)로 덮어써. (파일 자체는 남겨둠)
2. 그 직후, **정제된 프롬프트를 사용자의 새 요청으로 간주하고 실제 작업을 시작해.** (필요한 파일을 읽고, 코드를 수정하고, 명령을 실행하는 등 평소처럼 진행)
3. 작업을 시작할 때 한 줄로 "prompt.md 비웠고 작업 시작합니다." 라고 알려.

### 사용자가 "아니오"를 선택하면
1. `prompt.md`는 절대 건드리지 마.
2. "어떤 부분을 다시 다듬을까요?" 라고 한 줄로 물어봐.
3. 사용자가 피드백을 주면 정제 단계(섹션 2)부터 반복. 다시 검토 요청까지.

## 제약

- 사용자 응답 전에 `prompt.md`에 어떤 변경도 가하지 마.
- 정제 단계에서 새 파일을 만들거나 코드를 수정하지 마. 오직 `prompt.md` 읽기 + 사용자에게 보여주기까지만.
- 실제 작업은 "예" 응답 이후에만 시작.
